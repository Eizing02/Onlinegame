# MVP Checklist

ใช้ checklist นี้ก่อนนำไปทดลองในห้องเรียน

- [x] มี login รวมที่แยกครู/นักเรียนจากรหัสใน `data/student.csv`
- [x] ครูเข้าหน้าชุดคำถามได้
- [x] นักเรียนเข้าหน้าใส่รหัสห้องได้
- [x] ครูสร้างชุดคำถามได้จริง
- [x] ครูเพิ่มคำถาม คำตอบ คะแนน และเวลาต่อข้อได้จริง
- [x] ชุดคำถามถูกบันทึกใน `data/question-sets.json`
- [x] ครูสร้างห้องเกมจากชุดคำถามได้
- [x] ห้องเกมถูกบันทึกใน `data/game-sessions.json`
- [x] รหัสห้องเป็นตัวเลข 4 หลัก
- [x] นักเรียนเข้าห้องด้วย `room_code`
- [x] นักเรียนเลือกทีมเองหรือกดสุ่มทีมอัตโนมัติได้
- [x] ครูเห็นรายชื่อนักเรียนและทีมใน dashboard แบบ auto-refresh

## ก่อนเริ่มระบบเล่นจริง

- [x] หน้า join แสดงชื่อกิจกรรม/ชุดคำถาม/จำนวนทีมก่อนเลือกทีมเพื่อกันเข้าผิดห้อง
- [x] จำกัดหรือเตือนเมื่อครูมี active room อยู่แล้วก่อนสร้างห้องใหม่
- [x] เพิ่ม field `answers`, `lockedMemberCount`, `isScoreEligible`, `joinedAfterStart`, `lastSeenAt`

## เริ่มเกมและล็อกทีม

- [x] Dashboard ครูแสดงคำเตือนทีมว่าง/ทีมไม่ครบโดยไม่บล็อกปุ่มเริ่มเกม
- [x] ครูกดเริ่มเกมได้เมื่อมีนักเรียนอย่างน้อย 1 คน แม้ทีมไม่ครบ
- [x] ระบบล็อกจำนวนสมาชิกของแต่ละทีมตอนเริ่มเกมเพื่อใช้หารคะแนนเฉลี่ย
- [x] นักเรียนที่เข้าหลังเริ่มเกมเข้าได้แต่ไม่ร่วมคะแนนรอบนั้น
- [x] นักเรียน reconnect/refresh แล้วกลับเข้าห้องเดิม ทีมเดิม และสถานะล่าสุด

## คำถามและการตอบ

- [x] ระบบเปลี่ยนสถานะเกมตาม flow `lobby -> question_active -> answer_locked -> showing_answer -> ended`
- [x] ครูเริ่มคำถามข้อแรกได้
- [x] นักเรียนเห็นคำถามปัจจุบัน
- [x] นักเรียนส่งคำตอบได้
- [x] Server ตรวจคำตอบรายคนและกันตอบซ้ำในคำถามเดียวกัน

## คะแนนและสรุปผล

- [x] Server คิดคะแนนรวมรายทีมจากคำตอบถูกของสมาชิก
- [x] Server คิดคะแนนเฉลี่ยทีมด้วยสูตร `คะแนนรวม / จำนวนสมาชิกตอนเริ่มเกม`
- [x] Dashboard ครูเห็นคะแนนเฉลี่ยและอันดับแบบ auto-refresh
- [x] Dashboard/ผลสรุปรองรับกรณีคะแนนเท่ากันเป็นอันดับร่วม
- [x] มีหน้าสรุปผลหลังจบเกมเฉพาะรอบนั้น

## งานเสริมหลัง MVP เล่นได้

- [ ] ครูแก้ไขชื่อชุดคำถาม/แก้ไขคำถามเดิมได้

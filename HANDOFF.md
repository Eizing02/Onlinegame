# Handoff: Realtime Quiz Game

เอกสารนี้ใช้ส่งต่องานเมื่อย้ายไปทำต่อในเครื่องอื่น เพื่อให้ทิศทางยังเหมือนเดิม

## เป้าหมายของ MVP

สร้างเกมตอบคำถามออนไลน์แบบกลุ่มสำหรับใช้คั่นเวลาในห้องเรียน เข้าใช้ง่าย เล่นให้จบภายในรอบนั้น และเก็บถาวรเฉพาะชุดคำถาม

## แนวทางที่ตกลงกัน

- ใช้ login จาก `data/student.csv` โดยดู `role` เพื่อแยกครู/นักเรียน
- รหัสห้องเป็นตัวเลข 4 หลัก
- ครูสร้างห้องจากชุดคำถาม และกำหนดจำนวนทีม/จำนวนคนต่อทีมได้
- นักเรียน login, กรอกรหัสห้อง, เลือกทีมเอง หรือกดสุ่มทีม
- Dashboard ครูเห็นนักเรียนอยู่ในทีมแบบ auto-refresh
- ครูกดเริ่มเกมได้แม้ทีมไม่ครบ ขอแค่มีนักเรียนในห้องอย่างน้อย 1 คน
- ทีมว่างหรือทีมไม่ครบให้แสดงเป็นคำเตือนใน dashboard แต่ไม่บล็อกการเริ่มเกม
- ถ้านักเรียนเข้าหลังเริ่มเกม ให้เข้าดู/กลับเข้าห้องได้ แต่ไม่ร่วมคะแนนรอบนั้น
- ถ้านักเรียน refresh หรือ reconnect ให้กลับเข้าห้องเดิม ทีมเดิม และสถานะเกมล่าสุด
- ครูควรมี active room ได้ทีละ 1 ห้องใน MVP เพื่อลดโอกาสนักเรียนเข้าผิดห้อง
- คะแนนคิดจากรายคน แต่สรุปเป็นคะแนนเฉลี่ยรายกลุ่ม
- จำนวนสมาชิกที่ใช้หารคะแนนเฉลี่ยให้ล็อกตอนครูกดเริ่มเกม
- ถ้าคะแนนเฉลี่ยเท่ากัน ให้ถือว่ามีอันดับเสมอกัน
- ช่วง MVP ใช้ local JSON ก่อน ยังไม่เน้นความปลอดภัยซับซ้อน
- โหมด Supabase พร้อมใช้ผ่าน `DATA_BACKEND=supabase` หลังรัน migration และ seed ข้อมูล
- คำถามต้องเก็บไว้ใช้ซ้ำ ส่วนผลการแข่งขันเก็บเฉพาะรอบนั้น

## กติกาเริ่มเกมและคะแนน

### การเริ่มเกม

- ครูเริ่มเกมได้เมื่อมีนักเรียนอย่างน้อย 1 คนในห้อง
- ไม่จำเป็นต้องรอให้ทุกทีมเต็ม
- ทีมที่ไม่มีสมาชิกยังอยู่ในห้องได้ แต่ไม่ถูกนำไปคิดอันดับ
- ทีมที่สมาชิกไม่ครบยังเล่นได้ตามจำนวนคนที่มี
- Dashboard ครูควรแสดงสถานะเตือน เช่น `บางทีมยังไม่ครบ` หรือ `มีทีมว่าง` แต่ปุ่มเริ่มเกมยังใช้งานได้
- ตอนเริ่มเกมให้ snapshot สมาชิกที่อยู่ในแต่ละทีม และ mark เป็นผู้เล่นที่มีสิทธิ์ได้คะแนน
- หลังเริ่มเกมแล้ว นักเรียนที่เข้าทีหลังไม่เพิ่มตัวหารคะแนนเฉลี่ย

### Late join และ reconnect

- ถ้าเกมยังเป็น `lobby` นักเรียนเลือกทีม/เปลี่ยนทีมได้ตามกติกาห้อง
- ถ้าเกมเริ่มแล้ว นักเรียนที่ยังไม่เคยเข้ารอบนี้สามารถเข้าดูเกมได้ แต่ `isScoreEligible = false`
- นักเรียนที่เคยเข้าก่อนเริ่มเกมแล้ว refresh/reconnect ต้องกลับเข้าทีมเดิม และยังมีสิทธิ์คะแนนตามเดิม
- ถ้านักเรียน reconnect ระหว่างคำถามที่ยังเปิดอยู่ และยังไม่เคยตอบข้อนั้น ให้ตอบได้
- ถ้านักเรียน reconnect หลังตอบแล้ว หน้าเล่นต้องแสดงว่า `ส่งคำตอบแล้ว`
- เพิ่ม `lastSeenAt` เพื่อบอกสถานะ online/offline แบบง่ายใน dashboard

### ป้องกันเข้าผิดห้อง

- รหัสห้องยังเป็นตัวเลข 4 หลักเพื่อให้ใช้ง่าย
- ระบบต้องไม่สร้างรหัสซ้ำกับห้องที่ยัง active
- หน้า `/join` ต้องแสดงชื่อกิจกรรม ชุดคำถาม และจำนวนทีมก่อนให้นักเรียนเลือกทีม
- ใน MVP ให้ครู 1 คนมี active room ได้ทีละ 1 ห้อง ถ้าสร้างใหม่ควรเตือนหรือปิดห้องเก่า
- Dashboard ครูแสดงรหัสห้องและชื่อกิจกรรมชัดเจน

### การคิดคะแนน

นักเรียนตอบเป็นรายคน แต่คะแนนที่ใช้จัดอันดับเป็นคะแนนเฉลี่ยของทีม

```txt
คะแนนเฉลี่ยทีม = คะแนนรวมจากสมาชิกในทีมที่ตอบถูก / จำนวนสมาชิกในทีมตอนเริ่มเกม
```

ตัวอย่าง:

```txt
คำถามข้อละ 10 คะแนน

ทีม 1 มี 5 คน ตอบถูก 4 คน
คะแนนรวม = 40
คะแนนเฉลี่ย = 40 / 5 = 8

ทีม 2 มี 3 คน ตอบถูก 3 คน
คะแนนรวม = 30
คะแนนเฉลี่ย = 30 / 3 = 10
```

ข้อกำหนด:

- ตอบถูกได้คะแนนตามค่าของคำถาม
- ตอบผิดได้ 0 คะแนน
- ไม่มีคะแนนติดลบใน MVP
- นักเรียน 1 คนตอบได้ 1 ครั้งต่อคำถาม
- กุญแจกันตอบซ้ำคือ `roomCode + questionIndex + studentCode`
- คะแนนรวมดิบเก็บไว้ได้ แต่ dashboard และอันดับใช้คะแนนเฉลี่ย
- ถ้าคะแนนเฉลี่ยเท่ากัน ให้แสดงเป็นอันดับร่วม/เสมอ
- จำนวนสมาชิกที่ใช้หารต้องล็อกตอนเริ่มเกม เพื่อไม่ให้คะแนนเปลี่ยนถ้ามีนักเรียนเข้าเพิ่มทีหลัง

### สถานะเกมและคำถาม

ใช้ flow นี้เป็นแกน:

```txt
lobby
→ question_active
→ answer_locked
→ showing_answer
→ question_active ข้อถัดไป
→ ended
```

- `lobby`: รอนักเรียนเข้าห้องและเลือกทีม
- `question_active`: นักเรียนตอบคำถามปัจจุบันได้
- `answer_locked`: ปิดรับคำตอบแล้ว
- `showing_answer`: แสดงเฉลย
- `ended`: จบเกมและแสดงสรุปผล

## โครงข้อมูล gameplay ที่ใช้งานแล้ว

ใน `StoredTeam`:

```ts
lockedMemberCount: number | null;
rawScore: number;
averageScore: number;
```

ใน `StoredParticipant`:

```ts
joinedAfterStart: boolean;
isScoreEligible: boolean;
lastSeenAt: string;
```

ใน `StoredGameSession`:

```ts
startedAt: string | null;
endedAt: string | null;
answers: StoredAnswer[];
```

type คำตอบ:

```ts
type StoredAnswer = {
  id: string;
  questionIndex: number;
  studentCode: string;
  teamId: string;
  answerText: string;
  isCorrect: boolean;
  scoreAwarded: number;
  submittedAt: string;
};
```

## สิ่งที่ทำเสร็จแล้ว

- Login ครู/นักเรียนจาก `data/student.csv`
- ครูสร้างชุดคำถามและเพิ่มคำถามได้
- ครูสร้างห้องเกมพร้อมรหัสห้อง 4 หลัก
- ครูกำหนดจำนวนทีมและจำนวนคนต่อทีมได้
- นักเรียนเข้าห้องด้วยรหัส 4 หลักได้
- นักเรียนเลือกทีมเองได้
- นักเรียนกดสุ่มทีมให้ระบบเลือกทีมที่ว่างได้
- นักเรียนถูกพาไปหน้าเล่นและรอครูเริ่มเกม
- Dashboard ครูเห็นรายชื่อนักเรียนแยกตามทีม และดึงข้อมูลล่าสุดเอง
- หน้า join แสดงรายละเอียดห้องก่อนเลือกทีมเพื่อกันเข้าผิด
- ครูถูกจำกัด/เตือนเมื่อมี active room ที่ยังไม่จบ
- ครูกดเริ่มเกมได้เมื่อมีนักเรียนอย่างน้อย 1 คน แม้ทีมไม่ครบ
- ระบบล็อกจำนวนสมาชิกทีมตอนเริ่มเกมเพื่อใช้หารคะแนนเฉลี่ย
- นักเรียนเข้าหลังเริ่มเกมได้ แต่ไม่ร่วมคะแนนรอบนั้น
- นักเรียน refresh/reconnect แล้วกลับเข้าห้องเดิม ทีมเดิม และสถานะล่าสุด
- ครูควบคุมเกมได้: เริ่มเกม, ล็อกคำตอบ, เฉลย, ข้อถัดไป, จบเกม
- นักเรียนเห็นคำถามปัจจุบันและส่งคำตอบได้
- Server ตรวจคำตอบรายคนและกันตอบซ้ำในคำถามเดียวกัน
- Server คิดคะแนนรวมและคะแนนเฉลี่ยรายทีม
- Dashboard ครูเห็นคะแนนเฉลี่ยและอันดับแบบ auto-refresh
- Dashboard/ผลสรุปรองรับอันดับร่วมเมื่อคะแนนเฉลี่ยเท่ากัน
- มีหน้าสรุปผลหลังจบเกมเฉพาะรอบนั้น

## ขั้นถัดไป

1. สร้าง Supabase project จริง แล้วใส่ env ใน `.env.local`
2. รัน migration จาก `supabase/migrations/001_initial_schema.sql`
3. รัน `npm run seed:supabase` เพื่อนำบัญชีและชุดคำถามขึ้น Supabase
4. เปิด `DATA_BACKEND=supabase` แล้วทดสอบ flow จริง
5. ทดลองหลายเครื่องผ่าน Wi-Fi เดียวกันหรือ Cloudflare Tunnel
6. ทำระบบแก้ไขชื่อชุดคำถามและแก้ไขคำถามเดิม
7. เพิ่มการตั้งค่าเวลาอัตโนมัติ เช่นหมดเวลาแล้วล็อกคำตอบเอง
8. เพิ่ม export/พิมพ์ผลสรุปหลังจบเกมถ้าต้องการใช้งานในชั้นเรียนจริง
